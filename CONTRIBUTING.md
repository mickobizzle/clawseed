# Contributing to ClawSeed

Thanks for helping improve ClawSeed.

## Principles
- Keep content generic and reusable.
- Do not commit personal/private data.
- Prefer explicit, minimal changes.

## Workflow
1. Fork and create a branch.
2. Make focused edits.
3. Run local checks:
   - `./scripts/safety_scan.sh`
   - `./scripts/validate_seed.sh`
4. Open PR with a concise summary.

## Content boundaries
- No personal knowledge corpora.
- No contact lists, chat IDs, private logs.
- Use placeholders in templates.

## Commit style
- docs: for docs/templates
- feat: for scripts/features
- chore: maintenance/refactors
